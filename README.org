#+title: Cyclone V SoC: MIPS32

* Hardware-Software Bridge
This repository contains the hardware (VHDL) and software (C) co-design for a MIPS32 processor implementation on an Intel Cyclone V SoC FPGA (DE10-Standard). It establishes a headless, direct-connection workflow between a Linux development machine and the ARM Cortex-A9 Hard Processor System (HPS).
** Hardware Configuration & Quartus Compilation

To successfully route the hardwired HPS peripherals (DDR3, Ethernet, USB, UART) out to the physical board pins, the following rules must be maintained in Quartus/Platform Designer (Qsys):

- Top-Level Port Mapping: Every HPS "border" signal generated by Qsys (e.g., memory_mem_a, hps_io_emac1_inst_TX_CLK) must be explicitly mapped to the physical board pins in the top-level VHDL port map. They cannot be left open or mapped to internal signals.

- DDR3 Electrical Termination: To clear the SERIESTERMINATIONCONTROL and PARALLELTERMINATIONCONTROL fitter errors, the auto-generated Tcl script must be executed prior to final compilation:

- Tools -> Tcl Scripts -> hps_sdram_p0_pin_assignments.tcl -> Run

- JTAG Programming: The Cyclone V uses a daisy-chain JTAG configuration. When programming via the Quartus Programmer, the SOCVHPS (ARM core) device must be ignored. The .sof bitstream must only be attached and programmed to the second device, the FPGA fabric (5CSXFC6D6).
** Headless Linux & Networking Setup

The board runs a stripped-down, non-GUI Angstrom Linux distribution. To avoid relying on a USB serial console and external routers, the board is configured with a forced static IP over a direct Ethernet connection to the host laptop.

- Host Configuration: The laptop's Ethernet port is set to "Shared to other computers" (acting as a DHCP server/gateway at 10.42.0.1).

- Board Configuration (Bypassing ConnMan): Angstrom's default network manager (connman) conflicts with standard static IP configurations. To guarantee the board always boots with 10.42.0.50, a custom systemd service is injected:

#+begin_src bash
# /etc/systemd/system/eth0-static.service
[Unit]
Description=Force eth0 Static IP
After=network.target

[Service]
Type=oneshot
ExecStart=/sbin/ifconfig eth0 10.42.0.50 netmask 255.255.255.0 up
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target

#+end_src

SSH Alias: The host machine uses an ~/.ssh/config alias to simplify connection and file transfers:

#+begin_src bash
Host cyclone
    HostName 10.42.0.50
    User root
#+end_src

*** ⚠️ A Note on HPS SDRAM Pin Assignments
When Platform Designer (Qsys) generates the Hard Processor System (HPS), it also generates a specific Tcl script located at:
~fpga/soc_mips/synthesis/submodules/hps_sdram_p0_pin_assignments.tcl~

*Why is this needed?*
The Cyclone V HPS DDR3 memory controller requires strict electrical termination settings (=SERIESTERMINATIONCONTROL= and =PARALLELTERMINATIONCONTROL=) to map physically to the board. If these assignments are missing, the Quartus Fitter will fail.

*How to use it (The "Run Once" Rule):*
This script is *intentionally excluded* from the automated ~build.tcl~ pipeline. Running it simultaneously with the main compilation flow causes a Quartus database lock collision (=Error 125091: Can't run Tcl command while a process is in progress=). 

Because these settings are permanently written to your ~.qsf~ file the first time they are executed, you only need to run this script *once* under two conditions:
1. When setting up the repository for the very first time.
2. If you ever regenerate the ~soc_mips~ system in Platform Designer.

To run it manually, open the Quartus GUI, navigate to *Tools -> Tcl Scripts*, select ~hps_sdram_p0_pin_assignments.tcl~, and click *Run*. Once applied, you can safely use ~make fpga-build~ for all future command-line compilations.

** Hardware/Software Bridge Communication

Communication between the ARM processor (Linux user-space) and the FPGA fabric is handled via the Lightweight HPS-to-FPGA AXI Bridge.

- Base Address: The bridge is hardwired to physical address 0xFF200000.

- Memory Mapping: The C program uses open("/dev/mem") and mmap() to cast the physical hardware addresses into virtual memory pointers.

- Compilation: Because the code uses standard C99 loop declarations alongside POSIX extensions (like usleep), it must be compiled using the GNU dialect on the board:

#+begin_src bash
    gcc led.c -o led -std=gnu99
#+end_src
